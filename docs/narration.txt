Archon is a unified financial intelligence platform. It brings a business's whole financial picture — invoices, payments, bank statements, workforce costs and expenses — into one place, and gives an agent a memory that persists and is queryable across separate sessions. Everything here runs live on Alibaba Cloud, on real Qwen models.

First, proof it's real. One call to the health endpoint shows the embedder is Qwen text-embedding-v4 and the narrator is qwen-plus, producing genuine 1024-dimension embeddings. No stubs, no fakes.

Now the headline: a memory that audits itself. Accumulate facts across many sessions, and an agent eventually remembers one record two ways — and a plain vector store just returns whichever ranked higher and stays silent. Here, one record is written in two separate sessions: the first records the employer cost at eighteen thousand euros, a later write, nineteen thousand. We ask the agent to audit its own memory. It detects the contradiction — same record, same attribute, two conflicting values — and recommends which to trust: the later nineteen thousand, with a rule, a confidence, and a rationale. It's a recommender, not ground truth; it never mutates memory, it just flags when it disagrees with itself. And it's measured: five of five injected contradictions caught with zero false positives, four of four resolutions correct.

Next, the core of the MemoryAgent track — cross-session persistence. In Session A, the agent reconciles a bank confirmation, a payroll register and the payslips into one accurate event, and commits the key facts: the true cost of employing the team, and the gap the bank statement never shows. They're embedded and written to pgvector on Alibaba Cloud. Then Session A ends — the client disconnects, nothing stays in process, the facts live only in the database.

Session B is a fresh, later session: a different client, no shared state, only a question — what did it truly cost to employ the team, versus what left the bank? Qwen embeds the question, runs a cosine nearest-neighbour search over pgvector, and qwen-plus writes a grounded, cited answer: the true cost was fifteen thousand eight hundred euros against ten thousand that left the bank — a five thousand eight hundred euro gap. The scores are real cosine similarities: a memory written in one session, recalled by meaning in another.

The memory is genuinely queryable, not a canned reply: a second question — the cost of employing one named person — returns a fresh grounded answer from the same memory.

Finally, deployment. This runs on Alibaba Cloud: an ECS instance in Singapore running docker-compose — the backend beside a self-hosted pgvector container with an HNSW cosine index — plus text-embedding-v4 and qwen-plus from Model Studio. Self-auditing, persistent, queryable, across sessions. The MemoryAgent track, proven live.
