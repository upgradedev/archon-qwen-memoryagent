Archon is a unified financial intelligence platform — an agent memory that persists and is queryable across sessions, live on Alibaba Cloud on real Qwen models.

First, proof it's real: the health endpoint shows the embedder is Qwen text-embedding-v4 and the narrator qwen-plus, with genuine 1024-dimension embeddings — no stubs.

Now the headline: a memory that audits itself. One record is written in two separate sessions — first, employer cost eighteen thousand euros; a later write, nineteen thousand. We ask the agent to audit its own memory. It detects the contradiction — same record, two conflicting values — and recommends which to trust: the later nineteen thousand, with a rule, a confidence, and a rationale. It never mutates memory; it just flags when it disagrees with itself.

Next, the MemoryAgent core — cross-session persistence. In Session A, the agent fuses a bank confirmation, a payroll register and payslips into one event, and commits the key facts: the true cost of employing the team, and the gap the bank statement never shows. They're embedded and written to pgvector on Alibaba. Then Session A ends — the client disconnects, nothing stays in process.

Session B is fresh and separate: a different client, no shared state, only a question — what did it truly cost to employ the team, versus what left the bank? Qwen embeds the question, runs a cosine search over pgvector, and qwen-plus writes a grounded, cited answer: fifteen thousand eight hundred euros of true cost against ten thousand that left the bank. Real cosine similarities — a memory written in one session, recalled by meaning in another.

The memory is genuinely queryable: a second question — the cost of one named person — returns a fresh grounded answer from the same memory.

Finally, deployment: live on Alibaba Cloud — an ECS instance in Singapore running the backend beside a self-hosted pgvector container, with text-embedding-v4 and qwen-plus from Model Studio. Self-auditing, persistent, queryable, across sessions — proven live.
