**Live:** <https://memory.43.106.13.19.sslip.io>

**Operator gate:** the source candidate is not judge-ready until
[`deploy/DEPLOY_STATE.md`](../deploy/DEPLOY_STATE.md) records a successful exact

The public click path below needs **no login or API key**. It is intentionally limited to the fixed demo payload and public-tenant reads. scan. [`SUPPLY_CHAIN.md`](./SUPPLY_CHAIN.md) defines its pins and limitations;

## 60 seconds: seed, recall, self-audit

1. Open the [Memory Explorer](https://memory.43.106.13.19.sslip.io).
2. Click **Run demo**. The server idempotently seeds only its fixed payroll evidence plus planted field-level and meaning-level contradictions. It accepts no caller-controlled content. The seed is public but quota-bounded.
3. Click **Using only the retrieved memory, state the true employer cost for
   Northwind Trading in 2026-05 and include citation marker [1] in the
   sentence.** The Explorer requests a bounded top-three slice (up to three
   memories) from the public tenant and returns a grounding-validated answer
   with a resolvable `[1]` citation. Recall is also protected by per-IP plus
   global daily quotas.
4. Click **Run self-audit**. The deterministic, read-only audit should show `INV-5521 · amount` recorded as `8400` and `8900`, then recommend `8900` under the recency rule. It reports the disagreement; it does not overwrite either memory.

That screen demonstrates the Track 1 core: persistent, queryable memory across writes, bounded recall for limited context windows, and an explicit consistency mechanism.

## Public curl path

```bash
BASE=https://memory.43.106.13.19.sslip.io
QUESTION='Using only the retrieved memory, state the true employer cost for Northwind Trading '
QUESTION+='in 2026-05 and include citation marker [1] in the sentence.'

curl -fsS "$BASE/ready"
curl -fsS -X POST "$BASE/demo/seed"
curl -fsS -X POST "$BASE/recall" \
  -H 'content-type: application/json' \
  --data "$(printf '{"question":"%s","company":"Northwind Trading","limit":3}' "$QUESTION")"
curl -fsS -X POST "$BASE/consistency" \
  -H 'content-type: application/json' \
  -d '{"company":"Northwind Trading"}'
```

Expected field-level result: a contradiction for `INV-5521`, values `8400` and `8900`, with `resolution.recommendedValue = 8900` and `rule = "recency"`.

## 30 seconds more: authenticated meaning-level audit

The semantic route is intentionally protected because it invokes the heavier Qwen judge. In the Explorer, paste the dedicated Devpost reviewer credential into the password-type **Judge token (protected audit/lifecycle)** field, keep the demo company selected, and click **Run semantic audit**. Do not publish or screenshot the credential. Swagger's **Authorize** control is an alternative. If a terminal is necessary, use an interactive no-echo prompt so the value is absent from shell history and curl's command-line arguments:

```bash
BASE=https://memory.43.106.13.19.sslip.io
set +x
read -rsp 'Reviewer credential: ' TOKEN
printf '\n'

curl -fsS -X POST "$BASE/consistency/semantic" \
  -H @<(printf 'authorization: Bearer %s\n' "$TOKEN") \
  -H 'content-type: application/json' \
  -d '{"company":"Northwind Trading","kind":"insight"}'
unset TOKEN
```

The fixed demo contains *"always pays on time"* and *"chronically late"*. The response should expose a read-only semantic contradiction with model/completion provenance. The committed offline labelled benchmark measures the deterministic offline judge at **90% recall, 100% precision, and 0 false positives**; those figures are not presented as live-Qwen accuracy.

The same credential may be used to inspect authenticated tenant-scoped mutations such as `/ingest/invoice`, `/feedback`, `/consolidate`, and `/forget`. Lifecycle endpoints preview by default and require `confirm=true` before changing state.

## Release-bound evidence beyond the click path

The final gallery is generated by `scripts/capture_submission_gallery.py`, not by
hand-editing response values. Before it writes any final frame, the gate must:

- send an original synthetic payroll-register + bank-confirmation PNG pair through
  protected `/ingest/documents` `dryRun`, require response-reported `qwen-vl-max`
  and one fused event, then prove zero writes, unchanged reviewer count, and zero
  exact-marker residue;
- store explicit feedback in Session A and require a fresh Session B to recall,
  cite, and apply it—persisted state, not model-weight learning;
- preview exactly one feedback-superseded retention candidate, confirm exactly one
  audited deletion, preserve protected seed/correction state, and scrub the marker;
- label the human-control live frame honestly as **Defer only**; Accept/Override
  are separately tested protected actions and are not claimed by that capture.

The resulting `demo/gallery/CAPTURE_REVIEW.json` binds those gates and every final
artifact hash to the exact deployment evidence. It exists only after a successful
live capture; a compositor self-test is never judge evidence.

## Reproduce the evidence locally

The benchmarks replay committed fixtures with no API key or database. The full test command also includes real-DB slices, which skip explicitly when no integration database is supplied.

```bash
git clone https://github.com/upgradedev/archon-qwen-memoryagent
cd archon-qwen-memoryagent
npm ci
npm run bench -- --gate
npm run bench:consistency -- --gate
npm run bench:semantic -- --gate
npm run bench:resolution -- --gate
npm run test:docs
npm test
npm run coverage
```

Use the final immutable CI artifact for exact test and coverage values; real-DB
slices skip explicitly when `DATABASE_URL` is absent. Model seams use
deterministic Fakes in offline CI, while production is configured to fail closed
without real Qwen.
